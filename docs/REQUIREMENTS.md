# 요구사항 문서
## 코드 퀄리티
- 변수 선언시 const 만 사용한다.
- 함수(또는 메서드)의 들여쓰기 depth는 1단계까지만 허용한다.
- indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- 함수의 매개변수는 2개 이하여야 한다.
- 함수에서 부수 효과를 분리하고, 가능한 순수 함수를 많이 활용한다.
- else를 쓰지 않고, early return 을 활용한다.
- 3항 연산자를 쓰지 않는다.
- 함수 길이는 10라인을 넘지 않는다.
- 함수 매개변수는 2개 이하여야 한다.
- 줄여 쓰지 않는다(축약 금지).
- 클래스의 프로퍼티를 외부에서 직접 조작하지 않는다. (Tell, Don't Ask)
- HTTP Layer 와 도메인 로직을 분리하여 테스트할 수 있게 만든다.

## 🎯 기능 요구사항
- REST API를 직접 설계하고 구현합니다. >> REST API가 무엇인가?
- endpoint, HTTP 메서드, 응답 형식, 에러 코드는 스스로 결정합니다.
  - endpoint
  - HTTP 메서드
    - GET (200)
    - POST (201)
    - DELETE (204)
    - PATCH (204)
  - 응답 형식
    - JSON
  - 에러 코드
    - 404, 400, 500의 3개
- 주문, 총 결제 금액 계산 기능은 따로 구현하지 않습니다
- 상품 >> 클래스로 구현, 상품 정보를 속성으로 만들어둠, 상품 데이터는 Map으로 관리한다.
  - 상품이 담고 있는 정보 /products
    - 상품 id (productId)
    - 상품 이름 (name)
    - 상품 개별 가격 (price)
    - 상품 사진 (thumbnailUrl)
    - 상품의 재고 (quantity)
  - 상품 목록을 조회할 수 있다. GET /products
  - 상품을 추가할 수 있다. POST /products
  - 상품을 삭제할 수 있다. DELETE /products/:productId
  - 해당 상품이 장바구니에 있으면 함께 제거한다 >> 상품의 id를 장바구니에서 보고 삭제할 수 있어야 함

- 장바구니 /cart
  - 장바구니에 담긴 상품 목록을 조회할 수 있다. >> GET /cart
  - 장바구니 상품의 수량을 변경할 수 있다. >> PATCH /cart/:id >> 본문에 JSON { quantity: number }
  - 장바구니에 담긴 상품을 제거할 수 있다. >> DELETE /cart/:id
  - 장바구니가 담고 있는 정보
    - 장바구니 id
    - 상품 객체 (id, 상품명, price, quantity)

## 프로그래밍 요구사항
- API 설계
  - endpoint와 HTTP 메서드를 REST 원칙에 맞게 스스로 설계한다
  - HTTP 상태 코드(200, 204, 201, 400, 404, 500 등) 사용 기준을 정하고 그 이유를 설명할 수 있다
- 서버
  - Node.js + Express를 사용한다
  - TypeScript 로 구현한다.
  - 데이터는 DB 가 아니라 in-memory(배열 또는 Map)로 관리한다
  - HTTP 상태 코드를 의미에 맞게 사용한다 (200, 201, 400, 404 등)
  - FE와 연동을 위한 CORS 설정을 한다
- 테스트
  - 도메인 로직과 API 요청을 100% 테스트 합니다.
  - 도메인은 Jest, API 호출은 supertest 를 활용합니다.
  - TDD 로 구현합니다.
- 검증 (Validation)
  - 필수 필드 누락 시 에러를 응답한다
  - 존재하지 않는 상품/장바구니 아이템 요청 시 에러를 응답한다
  - quantity는 1 이상 99 이하의 정수여야 한다
  - price는 0보다 큰 숫자여야 한다
  - 상품명은 최대 100자이다

## 프로젝트 폴더 구조
- 계층형 분리
  - 기술적인 역할에 따라 폴더 구분
- 폴더 구조
  - routes/ : 엔드포인트 주소만 정의
  - controllers/ : HTTP 요청(req)을 받고 응답(res)을 보내는 역할 (HTTP Layer)
  - services/ : 실제 비즈니스 로직 (총액 계산, 장바구니 조건 확인 등 도메인 로직)
