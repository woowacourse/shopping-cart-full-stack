# REQUIREMENTS

## 🎯 기능 요구사항
아래 기능 요구사항을 바탕으로 REST API를 직접 설계하고 구현합니다.
- [x] endpoint, HTTP 메서드, 응답 형식, 에러 코드를 스스로 결정한다.
- [x] 주문, 총 결제 금액 계산 기능은 구현하지 않는다.

### 상품
- [ ] 상품 목록을 조회할 수 있다.
- [ ] 상품을 추가할 수 있다.
- [ ] 상품을 삭제할 수 있다.
- [ ] 삭제된 상품이 장바구니에 있으면 함께 제거한다.

### 장바구니
- [ ] 장바구니에 담긴 상품 목록을 조회할 수 있다.
- [ ] 장바구니 상품의 수량을 변경할 수 있다.
- [ ] 장바구니에 담긴 상품을 제거할 수 있다.

## ✅ 프로그래밍 요구사항

### API 설계
- [ ] endpoint와 HTTP 메서드를 REST 원칙에 맞게 설계한다.
- [ ] HTTP 상태 코드(200, 204, 201, 400, 404, 500 등) 사용 기준을 정하고 그 이유를 설명한다.

### 서버
- [ ] Node.js + Express 사용
- [ ] TypeScript로 구현
- [ ] 데이터는 DB가 아닌 in-memory(배열 또는 Map)로 관리
- [ ] HTTP 상태 코드를 의미에 맞게 사용한다 (200, 201, 204, 400, 404 등)
- [ ] FE와 연동을 위한 CORS 설정

### 테스트
- [ ] 도메인 로직과 API 요청을 100% 테스트
- [ ] 도메인은 Jest, API 호출은 supertest 활용
- [ ] TDD로 구현

### 검증
- [ ] 필수 필드 누락 시 에러 응답
- [ ] 존재하지 않는 상품/장바구니 아이템 요청 시 에러 응답
- [ ] quantity는 1 이상 99 이하의 정수
- [ ] price는 0보다 큰 숫자
- [ ] 상품명은 최대 100자
