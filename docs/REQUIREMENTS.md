# 장바구니 BE 1단계 요구사항

## 1. 상품 기능

### 1.1 상품 목록 조회

- 사용자는 등록된 상품 목록을 조회할 수 있다.
- `GET /products` 요청으로 상품 목록을 조회한다.
- 상품 목록 조회에 성공하면 `200 OK` 상태 코드를 응답한다.
- 상품 목록은 배열 형태로 응답한다.
- 등록된 상품이 없는 경우 빈 배열을 응답한다.

### 1.2 상품 추가

- 사용자는 새로운 상품을 추가할 수 있다.
- `POST /products` 요청으로 상품을 추가한다.
- 상품 추가 시 상품명, 상품 이미지, 가격, 재고를 입력해야 한다.
- 상품 추가에 성공하면 `201 Created` 상태 코드를 응답한다.
- 생성된 상품 정보를 응답한다.

#### 상품 추가 검증

- 상품명이 누락되면 `400 Bad Request`를 응답한다.
- 상품명은 최대 100자까지 허용한다.
- 상품명이 100자를 초과하면 `400 Bad Request`를 응답한다.
- 가격이 누락되면 `400 Bad Request`를 응답한다.
- 가격은 0보다 큰 숫자여야 한다.
- 가격이 0 이하이거나 숫자가 아니면 `400 Bad Request`를 응답한다.
- 재고가 누락되면 `400 Bad Request`를 응답한다.
- 재고는 0 이상 99 이하의 정수여야 한다.
- 재고가 0 미만, 99 초과, 또는 정수가 아니면 `400 Bad Request`를 응답한다.
- 상품 이미지가 누락되면 `400 Bad Request`를 응답한다.

### 1.3 상품 삭제

- 사용자는 상품을 삭제할 수 있다.
- `DELETE /products/:productId` 요청으로 상품을 삭제한다.
- 상품 삭제에 성공하면 `200 OK` 상태 코드를 응답한다.
- 존재하지 않는 상품을 삭제하려고 하면 `404 Not Found`를 응답한다.
- 삭제하려는 상품이 장바구니에 담겨 있다면 장바구니에서도 함께 제거한다.

## 2. 장바구니 기능

### 2.1 장바구니 상품 목록 조회

- 사용자는 장바구니에 담긴 상품 목록을 조회할 수 있다.
- `GET /cart` 요청으로 장바구니 상품 목록을 조회한다.
- 장바구니 상품 목록 조회에 성공하면 `200 OK` 상태 코드를 응답한다.
- 장바구니 상품 목록은 배열 형태로 응답한다.
- 장바구니에 담긴 상품이 없는 경우 빈 배열을 응답한다.

### 2.2 장바구니 상품 추가

- 사용자는 상품을 장바구니에 추가할 수 있다.
- `POST /cart` 요청으로 상품을 장바구니에 추가한다.
- 장바구니에 추가할 때 `productId`와 `quantity`를 입력해야 한다.
- 장바구니 상품 추가에 성공하면 `201 Created` 상태 코드를 응답한다.
- 존재하지 않는 상품을 장바구니에 추가하려고 하면 `404 Not Found`를 응답한다.
- 이미 장바구니에 담긴 상품을 다시 추가하려고 하면 기존 장바구니 상품의 수량을 변경하지 않고 `400 Bad Request`를 응답한다.

#### 장바구니 상품 추가 검증

- `productId`가 누락되면 `400 Bad Request`를 응답한다.
- `quantity`가 누락되면 `400 Bad Request`를 응답한다.
- `quantity`는 1 이상 99 이하의 정수여야 한다.
- `quantity`가 1 미만, 99 초과, 또는 정수가 아니면 `400 Bad Request`를 응답한다.

### 2.3 장바구니 상품 수량 변경

- 사용자는 장바구니에 담긴 상품의 수량을 변경할 수 있다.
- `PATCH /cart/:cartItemId` 요청으로 장바구니 상품 수량을 변경한다.
- 요청 body에는 변경할 `quantity`를 입력한다.
- 수량 변경에 성공하면 `200 OK` 상태 코드를 응답한다.
- 변경된 장바구니 상품 정보를 응답한다.
- 존재하지 않는 장바구니 상품의 수량을 변경하려고 하면 `404 Not Found`를 응답한다.

#### 장바구니 상품 수량 변경 검증

- `quantity`가 누락되면 `400 Bad Request`를 응답한다.
- `quantity`는 1 이상 99 이하의 정수여야 한다.
- `quantity`가 1 미만, 99 초과, 또는 정수가 아니면 `400 Bad Request`를 응답한다.

### 2.4 장바구니 상품 제거

- 사용자는 장바구니에 담긴 상품을 제거할 수 있다.
- `DELETE /cart/:cartItemId` 요청으로 장바구니 상품을 제거한다.
- 장바구니 상품 제거에 성공하면 `200 OK` 상태 코드를 응답한다.
- 삭제된 `cartItemId`를 응답한다.
- 존재하지 않는 장바구니 상품을 제거하려고 하면 `404 Not Found`를 응답한다.

| 장바구니 상품 제거      | DELETE | `/cart/:cartItemId` |

## 4. HTTP 상태 코드 사용 기준

### 4.1 성공 응답

- `200 OK`
  - 조회 요청에 성공했을 때 사용한다.
  - 기존 리소스의 일부를 수정하고, 수정된 데이터를 응답할 때 사용한다.
  - 리소스 삭제에 성공하고, 삭제된 리소스의 식별자를 응답할 때 사용한다.

- `201 Created`
  - 새로운 리소스 생성에 성공했을 때 사용한다.

### 4.2 실패 응답

- `400 Bad Request`
  - 필수 필드가 누락된 경우 사용한다.
  - 요청 값의 형식이 올바르지 않은 경우 사용한다.
  - 유효성 검증에 실패한 경우 사용한다.

- `404 Not Found`
  - 요청한 상품이 존재하지 않는 경우 사용한다.
  - 요청한 장바구니 상품이 존재하지 않는 경우 사용한다.

- `500 Internal Server Error`
  - 서버 내부에서 예상하지 못한 에러가 발생한 경우 사용한다.

---

## 5. 데이터 관리 요구사항

- 데이터는 데이터베이스를 사용하지 않고 in-memory로 관리한다.
- 상품 데이터는 서버 실행 중에만 유지된다.
- 장바구니 데이터는 서버 실행 중에만 유지된다.
- 서버가 재시작되면 모든 데이터는 초기화된다.

- 상품 삭제 로직을 테스트한다.
- 상품 삭제 시 장바구니에서도 함께 제거되는지 테스트한다.
- 장바구니 상품 추가 로직을 테스트한다.
- 장바구니 상품 수량 변경 로직을 테스트한다.
- 장바구니 상품 제거 로직을 테스트한다.
- 상품 관련 검증 로직을 테스트한다.
- 장바구니 관련 검증 로직을 테스트한다.

### 6.2 API 테스트

- `GET /products` 요청을 테스트한다.
- `POST /products` 요청을 테스트한다.
- `DELETE /products/:productId` 요청을 테스트한다.
- `GET /cart` 요청을 테스트한다.
- `POST /cart` 요청을 테스트한다.
- `PATCH /cart/:cartItemId` 요청을 테스트한다.
- `DELETE /cart/:cartItemId` 요청을 테스트한다.
- 정상 요청의 상태 코드와 응답 body를 테스트한다.
- 잘못된 요청의 상태 코드와 에러 응답을 테스트한다.

## 7. 코드 작성 요구사항

- Node.js와 Express를 사용한다.
- TypeScript로 구현한다.
- FE와 연동할 수 있도록 CORS를 설정한다.
- 변수 선언 시 `const`만 사용한다.
- 함수 또는 메서드의 들여쓰기 depth는 1단계까지만 허용한다.
- 함수의 매개변수는 2개 이하로 작성한다.
- 함수 길이는 10라인을 넘지 않도록 작성한다.
- `else`를 사용하지 않고 early return을 활용한다.
- 3항 연산자를 사용하지 않는다.
- 축약어를 사용하지 않는다.
- 클래스의 프로퍼티를 외부에서 직접 조작하지 않는다.
- HTTP Layer와 도메인 로직을 분리한다.
- 도메인 로직은 Express 요청, 응답 객체에 의존하지 않도록 작성한다.
- 가능한 순수 함수를 활용하고, 부수 효과를 분리한다.
