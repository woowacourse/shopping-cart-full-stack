# 요구사항 정리

## 공통

- 에러 핸들러
  - 404: 잘못된 자원을 요청한 경우
  - 500: 서버 프로세스는 살아 있는데 내부에서 예외가 난 경우.

## 상품

- [ ] 전체 상품목록을 조회할 수 있다.
  - 200: 데이터 조회 성공 후 조회 데이터 반환
  - 204: 데이터 조회에 성공했지만, 반환할 데이터 없음.

- [ ] 상품을 추가할 수 있다.
  - 추가해야하는 속성: 이름, 재고, 이미지, 가격
  - 가격은 0보다 큰 숫자여야 한다.
  - 재고는 1이상 99 이하 정수여야 한다.
  - 이름은 최대 100자이다.
  - 201: 상품 추가 성공
  - 400: 요청 데이터 검증 오류

- [ ] 상품을 제거할 수 있다.
  - id를 이용해 상품을 제거한다.
  - query parameter: id
  - 상품 제거 후, 장바구니에 있는 상품도 제거된다.
  - 204: 상품 및 장바구니에서 삭제 완료. 반환할 데이터 없음.
  - 404: 요청 데이터가 db에 없는 경우.

## 장바구니

- [ ] 장바구니에 담긴 상품 목록을 조회할 수 있다.
  - 200: 데이터 조회 성공 후 조회 데이터 반환
  - 204: 데이터 조회에 성공했지만 반환할 데이터 없음.

- [ ] 장바구니 상품의 수량을 변경할 수 있다.
  - id와 수량을 이용해 장바구니 상품의 수량을 변경한다.
  - 수량은 1개 이상 현재 재고 이하여야 한다.
  - 204: 상품 재고 수정 성공
  - 400: quantity 검증에 실패한 경우 (없거나, 숫자가 아니거나, 1보다 작은 경우)
  - 404: 요청 데이터가 db에 없는 경우.
  - 409: quantity 검증에 실패한 경우 (현재 재고보다 많은 경우)

- [ ] 장바구니에 담긴 상품을 제거할 수 있다.
  - id를 이용해 장바구니의 상품을 제거한다.
  - 204: 상품 제거 성공
  - 404: 요청 데이터가 db에 없는 경우.
