# 장바구니 풀스택 미션

## 요구사항

### 공통

- 프로젝트 초기 세팅을 한다.
  - Node.js와 Express 기반으로 서버를 구성한다.
- CORS를 설정한다.

### 상품

- 상품 목록을 조회할 수 있다.
  - 상품 목록 조회 API를 개발한다.
- 상품을 추가할 수 있다.
  - 상품 추가 API를 개발한다.
  - 상품명은 최대 100자이다.
  - 가격은 0보다 큰 숫자여야 한다.
  - 재고 수량은 1 이상 99 이하의 정수여야 한다.
  - 필수 필드가 누락되면 에러를 응답한다.
- 상품을 삭제할 수 있다.
  - 상품 삭제 API를 개발한다.
  - 존재하지 않는 상품을 삭제하면 에러를 응답한다.

### 장바구니

- 장바구니에 담긴 상품 목록을 조회할 수 있다.
  - 장바구니 목록 조회 API를 개발한다.
- 장바구니 상품의 수량을 변경할 수 있다.
  - 장바구니 상품 수량 변경 API를 개발한다.
  - 변경할 상품 수량은 0보다 큰 숫자여야 한다.
  - 상품 재고보다 더 많은 수량으로 변경하면 에러를 응답한다.
  - 장바구니에 존재하지 않는 상품의 수량을 변경하면 에러를 응답한다.
  - 필수 필드가 누락되면 에러를 응답한다.
- 장바구니에 담긴 상품을 삭제할 수 있다.
  - 장바구니 상품 삭제 API를 개발한다.
  - 장바구니에 존재하지 않는 상품을 제거하면 에러를 응답한다.

- 장바구니에 상품을 추가할 수 있다. (도메인 로직상 장바구니에 담는 로직만 존재, API는 아직 구현하지 않음)
  - 장바구니 상품 추가 API를 개발한다.
  - 추가할 상품 수량은 0보다 큰 숫자여야 한다.
  - 상품 현재 재고보다 더 많은 수량을 추가하면 에러를 응답한다.
  - 존재하지 않는 상품을 추가하면 에러를 응답한다.
  - 필수 필드가 누락되면 에러를 응답한다.

## 테스트 시나리오

### 상품

- [x] 상품 목록을 조회한다.
- [x] 상품을 1개 추가하면, 해당 상품이 상품 목록에 추가된다.
- [x] 상품명이 100자를 초과하면 에러를 응답한다.
- [x] 가격이 0 이하이면 에러를 응답한다.
- [x] 재고 수량이 1 이상 99 이하의 범위를 벗어나면 에러를 응답한다.
- [x] 필수 필드가 누락되면 에러를 응답한다.
- [x] 상품을 1개 삭제하면, 해당 상품이 상품 목록에서 삭제된다.
- [x] 상품을 삭제하면, 해당 상품이 장바구니에서도 함께 제거된다.
- [x] 존재하지 않는 상품을 삭제하면 에러를 응답한다.

### 장바구니

- [x] 장바구니 상품 목록을 조회한다.
- [ ] 장바구니에 상품을 1개 추가하면, 상품이 해당 수량만큼 장바구니에 추가된다.
- [ ] 상품 재고보다 더 많은 수량을 추가하면 에러를 응답한다.
- [ ] 상품 수량이 1 이상의 정수가 아니면 에러를 응답한다.
- [ ] 존재하지 않는 상품을 장바구니에 추가하면 에러를 응답한다.
- [ ] 필수 필드가 누락되면 에러를 응답한다.
- [x] 장바구니 상품의 수량을 1 증가시키면, 해당 상품의 주문 수량이 1 증가한다.
- [x] 장바구니 상품의 수량을 1 감소시키면, 해당 상품의 주문 수량이 1 감소한다.
- [x] 상품 재고보다 더 많은 수량으로 변경하면 에러를 응답한다.
- [x] 상품 수량을 1 이상의 정수가 아닌 값으로 변경하면 에러를 응답한다.
- [x] 장바구니에 존재하지 않는 상품의 수량을 변경하면 에러를 응답한다.
- [x] 장바구니 상품 수량 변경 시 필수 필드가 누락되면 에러를 응답한다.
- [x] 장바구니에 담긴 상품을 삭제하면, 해당 상품이 장바구니 상품 목록에서 삭제된다.
- [x] 장바구니에 존재하지 않는 상품을 제거하면 에러를 응답한다.
